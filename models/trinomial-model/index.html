Trinomial Option Pricing Model: Complete Guide to Enhanced Lattice Methods

Master trinomial option pricing for faster convergence than binomial. Complete guide to Boyle trinomial tree, probabilities, and implementation in Python and C++.

Quick Answer

Trinomial model adds middle node to binomial tree for faster convergence. Three moves: up (u), middle (1), down (d). Achieves O(1/N²) convergence vs O(1/N) for binomial.

Trinomial Model Formulas

Boyle Parameterization:
Δt = T/N
λ = σ√(3Δt) (spacing parameter)
u = e^(λ)
d = e^(-λ)
m = 1

p_u = (1/2)[(σ²Δt + μ²Δt²)/λ² + μΔt/λ]
p_d = (1/2)[(σ²Δt + μ²Δt²)/λ² - μΔt/λ]
p_m = 1 - p_u - p_d

Where μ = r - σ²/2

Kamrad-Ritchken Parameterization:
λ = √(3/2) (optimal)
u = e^(σ√(Δt)·λ)
d = 1/u

p_u = (e^(rΔt/2) - e^(-σ√(Δt)·λ))² / (4 sinh²(σ√(Δt)·λ))
p_d = (e^(rΔt/2) - e^(σ√(Δt)·λ))² / (4 sinh²(σ√(Δt)·λ))
p_m = 1 - p_u - p_d

How Trinomial Pricing Works

The trinomial model extends binomial by adding a middle node (price unchanged). This provides an extra degree of freedom, allowing the tree to match three moments (mean, variance, and often skewness). The convergence rate improves from O(1/N) to O(1/N²), meaning trinomial requires exponentially fewer steps for same accuracy. The additional node also allows better handling of barrier options since the tree can be more flexibly aligned with barrier levels.

Implementation Steps

1

Set time step Δt = T/N

2

Choose spacing λ = σ√(3Δt) or optimal λ = √(3/2)

3

Calculate u = e^(λ), d = e^(-λ), m = 1

4

Compute risk-neutral probabilities p_u, p_m, p_d

5

Build price tree (recombining: S_ij = S_0·u^j·d^(i-j))

6

Compute option payoffs at terminal nodes

7

Work backward: V_ij = e^(-rΔt)[p_u·V_{i+1,j+1} + p_m·V_{i+1,j} + p_d·V_{i+1,j-1}]

8

For American options, take max of continuation and exercise value

9

Return V_00 as option price

Where Trinomial Excels

Barrier Options

Better barrier alignment than binomial. Middle node allows finer barrier placement. More accurate for knock-in and knock-out options.

Lookback Options

Faster convergence for path-dependent options. Requires fewer steps to capture min/max accurately. More efficient than binomial.

Asian Options

Trinomial tree can be combined with moment matching for Asian option pricing. Better than binomial for fixed strike Asians.

High Precision Pricing

When need <0.1% accuracy, trinomial reaches precision with 200-500 steps versus 2000+ for binomial.

Convergence Comparison

Binomial Error

Error ∝ 1/N, oscillatory. 100 steps: ~0.5% error. 1000 steps: ~0.05% error. Requires 10x steps for 10x accuracy.

Trinomial Error

Error ∝ 1/N², smooth. 50 steps: ~0.1% error. 100 steps: ~0.025% error. Requires 3x steps for 10x accuracy.

Richardson Applied

Apply Richardson extrapolation to both: binomial improved to 1/N², trinomial to 1/N³. Combine N and 2N results.

Historical Development

Phelim Boyle introduced the trinomial tree in 1986, recognizing that three branches provide an additional degree of freedom for matching moments. Kamrad and Ritchken (1991) optimized the spacing parameter for better convergence. The trinomial tree remains popular for pricing barrier and other exotic options where binomial convergence is slow.

Moment Matching Conditions

Trinomial tree parameters must satisfy:
1. p_u + p_m + p_d = 1 (probability sum)
2. E[ΔS/S] = e^(rΔt) - 1 (drift matching)
3. E[(ΔS/S)²] = e^(2rΔt)(e^(σ²Δt) - 1) (variance matching)

With u·d = 1 (recombining condition) and m = 1, we have 3 equations for 3 unknowns (p_u, p_m, λ). Solution gives p_u, p_d, λ as above.

Frequently Asked Questions

Is trinomial always better than binomial?

For same number of steps, yes. For same computational budget, often yes because 50 trinomial steps = 200 binomial steps computationally, and 50 trinomial more accurate than 200 binomial.

How to choose λ (spacing parameter)?

Boyle: λ = σ√(3Δt) ensures positive probabilities. Kamrad-Ritchken: λ = √(3/2) (optimal). For barriers, adjust λ to align nodes with barrier level.

What if probabilities are negative?

Negative probabilities indicate parameter violation. Increase time steps (Δt smaller) or switch to alternative parameterization. Occurs for extreme parameters (very high σ or r).

Can trinomial handle multiple assets?

Yes, but node count grows as 3^d (exponential). For d=2, 9 branches per node. For d>2, Monte Carlo becomes more efficient.
