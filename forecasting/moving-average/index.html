Quick Answer

Formula

Simple Moving Average (SMA):
F_t+1 = (Y_t + Y_t-1 + ... + Y_t-n+1) / n

Weighted Moving Average (WMA):
F_t+1 = (w1*Y_t + w2*Y_t-1 + ... + wn*Y_t-n+1) / (w1 + w2 + ... + wn)
where w_i are weights (typically w1 > w2 > ... > wn)

Where:
F_t+1 = Forecast for next period
Y_t = Actual value at time t
n = Window size

How Moving Average Forecasting Works

Moving average forecasting smooths out short-term fluctuations to reveal longer-term trends. The method assumes that future values will be similar to the average of recent observations. For SMA, each observation in the window contributes equally. For WMA, recent observations have higher weight, making the forecast more responsive to recent changes. The choice of window size is critical: small windows (3-5 periods) react quickly but produce noisy forecasts, while large windows (15-30 periods) produce smooth forecasts that lag behind changes.

Use Cases by Industry

Inventory Management

Forecast demand for products with stable, non-seasonal patterns. Window size typically 4-8 weeks for fast-moving items, 8-12 weeks for slower items.

Financial Trading

SMA and WMA for technical analysis. Short windows (5-20 days) for trading signals, longer windows (50-200 days) for trend identification.

Call Center Planning

Forecast call volumes for staffing. Use 4-6 week windows with weighting for recent trends. Helps optimize agent schedules.

Energy Demand

Predict electricity or gas consumption. Window size depends on seasonality - use separate models for different seasons.

Pros and Cons

Advantages

  • -Simple to understand and implement
  • -Computationally efficient (O(n) per forecast)
  • -Works well for stable, trend-free data
  • -No complex parameter estimation required
  • -Easy to update with new observations

Limitations

  • -Does not handle trends or seasonality well
  • -All past observations equally weighted in SMA
  • -Lagging indicator by definition
  • -Poor performance on volatile or cyclical data
  • -No built-in confidence intervals

Window Size Selection Guide

Window Size 2-5

Very responsive, noisy forecasts. Best for highly volatile data where recent changes dominate.

Window Size 6-10

Good balance for weekly data. Suitable for short-term forecasting with moderate volatility.

Window Size 11-20

Smooth forecasts with noticeable lag. Best for stable data where noise reduction is priority.

Window Size 21+

Very smooth, significant lag. Use only for very stable long-term trends or baseline estimation.

History of Moving Average

Moving averages have been used in forecasting since the early 20th century. The method gained prominence in inventory management during World War II. In the 1970s, moving averages became a cornerstone of technical analysis through the work of Robert Levy and others. Today, moving averages remain one of the most widely used forecasting methods due to their simplicity and interpretability.

Mathematical Derivation

The moving average forecast minimizes the sum of squared errors for a constant model. For SMA, the optimal forecast is the sample mean of the window. For WMA, optimal weights are proportional to the inverse of the error variance for each lag, with more recent observations typically having lower variance in many real-world datasets.

Frequently Asked Questions

How do I choose between SMA and WMA?

Use SMA when all past observations in the window are equally relevant. Use WMA when recent observations are more predictive, such as in trending markets or fast-changing environments.

What is the difference between moving average and rolling average?

They are the same thing. Moving average, rolling average, and running average all refer to calculating averages over a sliding window of observations.

Can moving average handle missing data?

No, moving average requires complete data. Missing values should be imputed (using interpolation or previous values) before applying moving average forecasting.
